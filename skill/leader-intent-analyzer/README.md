# 领导意图分析助手（Leader Intent Analyzer）

> 通过历史发言反向分析领导性格与决策倾向，在多领导环境中做出最优沟通决策。

## 简介

职场中最难的往往不是工作本身，而是**理解领导的真实意图**。

这个 skill 帮助你：
- **建立领导画像**：通过历史发言分析性格、决策风格、沟通偏好
- **分析真实意图**：当领导下发任务时，判断她真正想要什么
- **生成应对策略**：针对具体场景给出最优回复话术
- **管理多领导关系**：支持多个领导画像隔离管理，跨画像对比优化策略
- **持续迭代进化**：画像越用越精准，伴随你的职场成长

## 核心功能

| 功能 | 说明 |
|------|------|
| **多画像隔离管理** | 每个领导独立画像文件，完全隔离不串台 |
| **持续迭代更新** | 新素材自动追加，保守修正原则，画像越用越精准 |
| **跨画像对比** | 对比不同领导风格差异，优化多领导环境下的沟通策略 |
| **心理学深度分析** | 基于大五人格、依恋类型、压力传导、LMX理论等模型 |
| **场景化话术生成** | 针对紧急任务、公开批评、需求模糊等场景生成定制话术 |

## 适用人群

- 刚入职新环境，需要快速理解领导风格
- 同时向多个领导汇报，需要平衡不同偏好
- 与领导沟通有障碍，经常踩雷或误解意图
- 希望从"执行者"升级为"协作者"的职场人

## 快速开始

### 第一步：建立领导画像

把你收集到的领导发言记录发给我：

```
新建领导画像：产品组组长

职位：产品组组长
管理范围：产品小组（5人）

历史素材：
[2026-05-15 群里] "大家让大模型生成下各自的月报"
[2026-05-16 群里] "@我 以后每周五定一下产研测周例会"
[2026-05-20 私聊] "这个交互对，但这个图标字要考虑下吧"
...
```

系统会输出：
- 决策风格分析（果断/犹豫、数据驱动/直觉）
- 沟通风格分析（直接/委婉、时间敏感度）
- 核心价值观识别（最看重的关键词、底线雷区）
- 管理风格判断（授权程度、反馈频率）
- 心理学深度解读（人格特质、压力模式、关系动力学）

### 第二步：分析领导意图

当领导下发新任务时：

```
切换领导：产品组组长

她刚才说："@我，类型转换的梳理多少了？"
场景：晚上20:39，钉钉私聊
上下文：今天下午刚分配的任务
```

系统会输出：
- 显性需求（她说了什么）
- 隐性需求（她真正想要什么）
- 情绪状态（焦虑/平静/不满）
- 真实时间预期（"尽快"=多久）
- 应对策略（稳妥型/积极型/缓冲型）
- 推荐话术（3个版本）

### 第三步：持续更新画像

每次有新的互动，都可以更新画像：

```
更新产品组组长的画像

新增素材：
[2026-06-16 群里] "没有完成任务的，第一次100，还没完成要去找hr谈~"
[2026-06-16 私聊] "那你就弄完几个放几个，又不是说要一次性都放上"
```

系统会：
- 追加素材到历史素材库
- 对比新旧素材，识别变化点
- 更新画像分析（标记变更部分）
- 写入版本记录
- 输出更新摘要

### 第四步：跨画像对比（可选）

当你需要同时应对多个领导时：

```
对比产品组组长和技术总监

场景：我需要推动一个AI功能上线，需要两人同时批准
我的方案：引入AI智能推荐模块，预计开发周期2周
```

系统会输出：
- 两人风格差异对比表
- 各自预期反应和质疑点
- 推荐话术（针对各自偏好定制）
- 沟通顺序建议
- 风险预案

## 使用示例

### 示例1：紧急任务应对

**场景**：领导在群里@你，要求"尽快"完成一个需求

**输入**：
```
用产品组组长的画像

领导原话："@我，这个需求尽快给我一版，我要给老板汇报"
场景：周二下午3点，钉钉项目群
```

**输出**：
```
## 意图分析

### 显性需求
她需要这个需求的分析版本，用于向上汇报

### 隐性需求
- 她可能面临上级压力（"给老板汇报"）
- 她需要确定性（什么时候能给她）
- 她需要可展示的成果（不是草稿，是能汇报的）

### 真实时间预期
- 表面："尽快"
- 实际：基于她的画像，她可能希望今天下班前或明天上午

## 应对策略

### 方案A：稳妥型（推荐）
"收到，我立即着手。预计在今晚8点前给您初版，包含核心分析。您看可以吗？"

### 方案B：积极型
"收到，我优先级最高的处理。预计6点前给您框架，8点前给完整版。需要我重点突出哪个方面？"

### 方案C：缓冲型
"收到，这个需求涉及XX方面，为了确保质量，我需要确认两点：1.汇报的重点是... 2.格式要求是... 确认后我今晚给您。"
```

### 示例2：公开批评应对

**场景**：领导在群里公开质疑你的工作

**输入**：
```
用产品组组长的画像

领导原话："@我，这个bug怎么又出现了？"
场景：群里，还有其他同事在
```

**输出**：
```
## 应对策略

### 第一步：降温（立即回复，群里）
"收到，这个问题确实是我考虑不周，我立即排查修复，今天内给您更新。"

### 第二步：私下沟通（1小时内）
"领导，关于今天的问题，我复盘了一下，主要是在XX环节没考虑到。我已经在修复了，预计X点完成。完成后我详细跟您汇报原因和改进措施。"

### 第三步：重建信任（后续）
- 超预期交付：不仅修复bug，还补充了回归测试
- 增加汇报频率：主动同步进展

### 禁忌
- 不要在群里解释"因为..."（像在找借口）
- 不要沉默超过2小时
```

## 安装方法

### SOLO 原生安装

1. 将整个 `leader-intent-analyzer` 文件夹复制到你的 SOLO skills 目录
2. 重启 SOLO 即可识别

### Cursor 安装

1. 将 `SKILL.md` 的内容复制到 `.cursorrules` 文件中
2. 将 `references/` 下的内容作为上下文参考文件
3. 在对话中直接描述场景即可使用

### VSCode + Claude Code 安装

1. 将 `SKILL.md` 复制到 `.claude/commands/leader-intent-analyzer.md`
2. 在对话中输入 `/leader-intent-analyzer` 即可调用

## 核心设计原则

### 1. 保守修正原则
新素材与旧画像矛盾时，优先保留旧画像，标记"待验证"。同一新特征出现3次以上才正式更新。

### 2. 用户主导原则
重大画像变更（如从"授权型"改为"集权型"）需用户确认，系统不擅自覆盖。

### 3. 隐私保护原则
所有画像文件本地存储，不上传云端。用户可随时删除或迁移。

### 4. 伦理边界原则
本 skill 旨在改善职场沟通效率，不应用于操纵、恶意目的或侵犯他人隐私。

## 心理学模型支撑

本 skill 的分析框架基于以下心理学与管理学理论：

| 理论 | 应用场景 |
|------|---------|
| **大五人格模型（OCEAN）** | 分析领导的基本人格特质 |
| **依恋类型理论** | 判断领导的焦虑触发点和安全感需求 |
| **压力传导模型** | 理解中层管理者的行为动机 |
| **领导-成员交换理论（LMX）** | 评估你与领导的关系阶段 |
| **心理契约理论** | 识别隐性期望，避免信任破裂 |
| **非暴力沟通（NVC）** | 设计困难话题的沟通话术 |
| **情绪智力（EQ）** | 管理自己和领导的情绪反应 |
| **期望理论（Vroom）** | 设计让领导愿意支持你的激励逻辑 |

## 注意事项

1. **概率性判断**：意图分析是推测，标注置信度，用户需结合实际判断
2. **动态迭代**：领导风格会变化，建议持续补充新素材更新画像
3. **文化差异**：不同企业文化和行业有差异，分析结果需结合具体环境理解
4. **关系双向**：画像分析是工具，真正的信任建立需要持续的高质量互动
