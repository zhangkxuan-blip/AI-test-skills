# 使用示例：输入格式参考

## 示例1：建立领导画像

```markdown
### 领导基本信息
- 职位：产品总监
- 管理范围：产品部（15人）+ 设计部（8人）
- 公司背景：互联网中厂，B端SaaS业务
- 入职时间：2年

### 历史发言记录

**2024-01-15 部门周会**
"这个季度的核心指标是客户留存率，不要只盯着新签。销售那边催得急，但我们要有定力，产品体验做扎实了，续费自然上来。"

**2024-02-03 钉钉私聊**
"那个需求评审的材料太粗了，数据呢？竞品分析呢？下次没有数据支撑的方案不要拿到会上来。"

**2024-02-20 项目群**
"@小王 @小李 这个需求今天必须上线，客户那边已经催了三次了。有问题随时找我，我今晚都在。"

**2024-03-05 一对一沟通**
"你最近表现不错，那个重构项目推进得很快。不过我要提醒你，不要只埋头干活，多想想业务价值，技术方案再优雅，不能解决业务问题也是白搭。"

**2024-03-12 跨部门会议**
"我理解运营那边的压力，但产品有自己的节奏。这样，我们折中一下，先上一个MVP版本，两周后看数据再决定要不要继续投入。"

**2024-03-20 年会发言**
"过去一年我们取得了很多成绩，但我最骄傲的不是数据，是团队。看到大家成长，比我自己的晋升还开心。"

**2024-04-01 钉钉群**
"这个设计稿不行，太保守了。我们要有突破性，参考一下Notion和Figma的做法，不要老是抄竞品。"

**2024-04-10 批评场景**
"这个bug怎么又出现了？上次不是说过要加回归测试吗？我不想再看到同样的问题第二次。"

### 补充观察
- 经常在晚上10点后发消息
- 喜欢发长语音（1-3分钟）
- 群里发言多，私聊较少
- 对设计很挑剔，经常亲自改UI细节
- 和销售部关系一般，经常抱怨销售过度承诺
```

## 示例2：分析新任务意图

```markdown
### 领导原话
"@小张 那个数据分析的需求，尽快给我一版，我要给老板汇报。"

### 场景信息
- 沟通渠道：钉钉项目群（群里还有5个同事）
- 时间：周二下午3点
- 上下文：上周刚开完季度规划会，老板要求各部门提交数据复盘
- 历史：上个月领导提过想做数据看板，但一直没排期

### 我的顾虑
- "尽快"是多快？
- 需要到什么精度？Excel还是可视化图表？
- 是给老板看的汇总版，还是详细分析版？
```

## 示例3：请求回复话术

```markdown

### 场景信息
- 沟通渠道：邮件回复
- 时间：周五下午5点
- 上下文：我已经提交了第三版方案
- 领导画像已知：数据驱动型，直接型，对"深入"要求高

### 我的需求
- 想知道"不够深入"具体指什么
- 不想盲目改第四版
- 希望周末能确定方向，下周一开始执行
```

- 参会：产品总监（我领导）、CTO、CEO
- 议题：下半年产品路线图

### 我要汇报的内容
- 建议投入AI功能模块
- 预计需要增加2个开发人力
- 竞品已经在做类似功能

### 领导画像已知

## 输入建议

为了获得最佳分析效果，请尽量提供：

1. **完整上下文**：不要只给一句话，给前后背景
2. **具体场景**：什么场合、什么时间、还有哪些人在场
3. **历史对比**：领导以前对类似事情的态度
4. **你的目标**：你想通过这次沟通达成什么
5. **你的顾虑**：你担心什么、不确定什么

信息越完整，分析越精准。
